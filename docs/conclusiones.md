## Conclusiones
---
---

Este proyecto pretende convertirse en una plataforma centrada en una comunidad, combinando contenido, interacción entre usuarios y funcionalidades adicionales como minijuegos o reseñas. Aún no está comenzado.

---

## Dificultades previstas

### Coordinación del trabajo
La organización del proyecto, la gestión del tiempo y la planificación de tareas pueden suponer un reto, especialmente al integrar diferentes tecnologías y apartados.

### Gestión de la base de datos
El uso de MySQL para almacenar información estructurada y Firebase para funciones como autenticación o datos en tiempo real puede requerir una buena planificación para evitar inconsistencias y mantener la seguridad.

### Integración de tecnologías externas
La posible inclusión de minijuegos creados con Godot añade complejidad técnica, especialmente en lo referente a su integración dentro de una plataforma web.

---

## Aprendizajes esperados

### Uso de Git en equipo
El proyecto permitirá mejorar el control de versiones, la organización del repositorio y la colaboración mediante ramas, commits y revisiones.

### Documentación técnica en Markdown
La creación de esta documentación ayudará a desarrollar una estructura clara, ordenada y fácil de mantener, utilizando Markdown como herramienta principal.

---

## Posibles mejoras futuras

### Autenticación avanzada
En fases posteriores sería interesante implementar sistemas de autenticación más completos, como inicio de sesión con proveedores externos o verificación en dos pasos.

### Despliegue en la nube
La plataforma podría beneficiarse de un despliegue en servicios como Firebase Hosting, Vercel o AWS, lo que permitiría mayor escalabilidad y disponibilidad.
